# 색 제안 기능에 색 공간 변환이 필요한 이유

`--suggest` 기능은 WCAG 명암비를 충족하지 못하는 전경색을 자동으로 조정한다. 이 과정에서 sRGB → OkLCH 변환이 필수적인 이유를 세 가지 관점에서 설명한다.

## 1. hex 값은 감마 인코딩된 값이다

`#777777`과 같은 hex 표기는 sRGB 색 공간의 값이며, **감마 인코딩**이 적용되어 있다. 감마 인코딩은 인간의 눈이 어두운 영역의 밝기 차이에 더 민감한 특성을 활용하여, 어두운 영역에 더 많은 숫자 범위를 할당하는 비선형 변환이다.

sRGB의 감마 함수(IEC 61966-2-1)는 단순한 거듭제곱이 아니라 분할 함수(piecewise function)이다:

```text
값 ≤ 0.04045 일 때:  선형 = 값 / 12.92
그 외:              선형 = ((값 + 0.055) / 1.055) ^ 2.4
```

`#777777`을 예로 들면:

| 단계 | 값 |
|------|-----|
| hex → 정수 | 119 |
| 0–1 정규화 | 119 / 255 ≈ 0.467 |
| 감마 디코딩 (선형) | ≈ 0.184 |

0.467과 0.184는 큰 차이다. 감마 인코딩된 값을 그대로 수학 연산(휘도 계산, 색 보간 등)에 사용하면 **물리적으로 틀린 결과**가 나온다. WCAG의 상대 휘도 공식도 반드시 선형 RGB로 디코딩한 뒤 적용해야 정확하다.

## 2. 명도만 조정하려면 축이 분리된 색 공간이 필요하다

suggest 기능의 핵심 요구사항은 **"원래 색의 느낌(색조, 채도)을 최대한 살리면서 명암비만 맞추는 것"**이다. 이를 위해서는 명도를 독립적으로 조절할 수 있는 색 공간이 필요하다.

### sRGB에서 직접 조정하면 안 되는 이유

sRGB에는 **독립적인 밝기 축이 없다**. 세 채널을 동일 비율로 스케일링하면 채널 비율은 유지되지만 검정 방향으로 이동하며, 지각적 채도가 함께 변한다. 채널별로 다르게 조정하면 색조가 변한다. 어떤 방식이든 "밝기만 바꾸기"가 불가능하다.

### HSL도 부적합한 이유

HSL은 H(색조), S(채도), L(명도)로 축이 분리되어 있지만, **지각적으로 균일하지 않다**. HSL에서 L=50%인 파란색과 L=50%인 노란색은 인간의 눈에 전혀 다른 밝기로 보인다. HSL의 L을 동일하게 조정해도 지각적 밝기 변화량이 색마다 크게 달라서, 명암비 목표에 정확히 도달하기 어렵다.

### OkLCH가 적합한 이유

OkLCH는 OkLab의 극좌표 표현으로, L(명도), C(채도), H(색조)가 분리되어 있고, **지각적으로 균일하다**. L 값의 동일한 변화가 어떤 색이든 비슷한 밝기 변화로 느껴진다. 따라서:

- C와 H를 고정하고 **L만 이진 탐색**하면 색조와 채도를 보존하면서 목표 명암비를 달성할 수 있다.
- 지각적 거리가 최소화되므로, 제안된 색이 원래 색과 가장 비슷하게 느껴진다.

## 3. 변환 파이프라인

suggest 기능이 사용하는 전체 변환 경로는 다음과 같다.

### 전경색 → OkLCH (탐색 준비)

```text
sRGB (감마 인코딩, hex 입력값)
  ↓  감마 디코딩 (IEC 61966-2-1)
Linear sRGB
  ↓  3×3 행렬 변환
XYZ-D65 (절대 색 공간)
  ↓  OkLab 변환 (LMS 경유)
OkLab (지각 균일 색 공간)
  ↓  직교 → 극좌표 변환
OkLCH (L, C, H)
```

구현: `src/convert/srgb-to-oklch.ts`

### 이진 탐색 반복 (L 후보 → 휘도 평가)

```text
OkLCH(L_candidate, C, H)
  ↓  gamutMapOklch (sRGB 색역 매핑)
sRGB (감마 인코딩)
  ↓  alphaComposite (알파 합성)
sRGB (불투명)
  ↓  감마 디코딩
Linear sRGB
  ↓  ITU-R BT.709 가중치 적용
상대 휘도 → 목표 휘도와 비교
```

이진 탐색의 각 반복마다 이 파이프라인이 실행되어, 후보 L 값이 목표 휘도에 얼마나 가까운지를 평가한다.

## 요약

| 문제 | 색 공간 변환이 해결하는 것 |
|------|---------------------------|
| hex 값은 감마 인코딩되어 있다 | 선형 RGB로 디코딩해야 정확한 휘도 계산 가능 |
| sRGB에서는 밝기만 독립 조정 불가 | OkLCH에서 L/C/H 축 분리로 해결 |
| HSL은 지각적으로 불균일하다 | OkLCH의 지각 균일성으로 최소한의 조정으로 목표 도달 |
